prject: voting application
What?
A functionality where user can give vote to the given set of the candidates. 

Models?
Routes?

functionality: 
1. User can Signup/Signin
2. See the list of candidates
3. Vote one of the candidates (provide your vote only one person/candidate)
4. There is a route which shows the list of the candidates and their live voting count 
5. user data must contain their nid number and userInfo
6. There should be one admin who cannot provide his vote. who maintain the whole server and candidates. 
7. user can change their password
8. user can login only nid number and password. 

End points
--------------------------------------------------------

Routes

User Authentication: 
    /signup: POST - Create a new user account.
    /login: POST - Login to an existing account. (NID card number + password)

voting: 
    /candidates: GET - get the list of candidates
    /vote/:candidateId: POST - vote for a specific candidate 

Vote Counts: 
    /vote/count: GET - get the list of candidates sorted by their vote Counts

User Profile: 
    /profile: GET - get the user profile information 
    /profile/password: PUT - Changes the users password

Admin candidate Management: 
    /candidates: POST - Create a new candidate. 
    /candidates/:candidateId: PUT - Update an existing candidate. 
    /candidates/:candidateId: DELETE -Delete a candidate from the list.
